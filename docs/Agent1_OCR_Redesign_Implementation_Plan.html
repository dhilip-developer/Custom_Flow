1. Overview & Core Insight

Agent 1 is the OCR extraction engine of the customs pipeline. It receives a PDF upload (which may contain necessary customs documents mixed with unnecessary pages) and must extract text only from the relevant documents — Bill of Lading, Invoice, Packing List, Freight Certificate, and Insurance Certificate.

Agent 1 uses Google Document AI exclusively for all OCR processing. No Gemini AI is used in this agent — Gemini quota is reserved entirely for Agents 2–5 (classification, extraction, freight detection, cross-verification).

🧠 Core Insight Document AI already returns per-page text natively. Each document.pages[i] contains paragraphs, blocks, lines with textAnchor offsets into the main document.text. We don't need to split the PDF into single pages for API calls — we just need to read the response intelligently.

The architecture follows a "OCR First, Classify Second" principle: the expensive network-bound OCR runs in parallel chunks, then the cheap CPU-bound classification runs sequentially in milliseconds with zero additional API calls.

3. Phase-by-Phase Specification

Phase 1 — Parallel Chunk OCR

AttributeDetail
APIGoogle Document AI — OCR Processor (sync/online mode)
Authgoogle-credentials.json service account
Chunk size5 pages per chunk (safely under 15-page sync limit)
Parallelism3 concurrent ThreadPoolExecutor workers
Retry policyExponential backoff: 2n seconds, max 3 retries on 503/429
InputPDF file bytes + MIME type
OutputList of Document AI response objects — each containing document.text and document.pages[]

How it works:

  1. Read total page count via pypdf.PdfReader
  2. If ≤5 pages → send as single request (no splitting)
  3. If >5 pages → split into 5-page chunks using PdfWriter
  4. Submit all chunks to Document AI in parallel (3 workers)
  5. Collect responses, ordered by original page position

Phase 2 — Per-Page Text Extraction

AttributeDetail
CostZero — pure CPU operation
Time~5ms for 100 pages
InputDocument AI response objects from Phase 1
OutputList[PageResult] — one entry per physical page with page_number, text, first_lines

How it works:

  1. For each response, iterate document.pages[]
  2. For each page, use page.layout.textAnchor.textSegments[].startIndex / endIndex to slice the page's text from document.text
  3. Extract the first 5 lines as first_lines (used for title detection)
  4. Build PageResult objects with global page numbering

Phase 3 — Sequential Title Scan

AttributeDetail
CostZero — pure regex, no API calls
Time~2ms for 100 pages
InputList[PageResult] from Phase 2
OutputPages grouped into document segments with type labels

Decision logic (applied sequentially, page by page):

  1. Title matches a necessary document type → Start a new document group. Flush any previous group.
  2. No title found → Check if the previous page was part of a relevant group. If yes, this is a continuation page — append to that group. If no, skip.
  3. Title matches a junk pattern → Skip the page. Break the continuation chain so the next untitled page doesn't inherit a junk classification.

Phase 4 — Merge & Return

AttributeDetail
CostZero — string concatenation
Time~1ms
InputGrouped page segments from Phase 3
OutputSmartExtractionResponse JSON — documents grouped by type + skipped sections list

How it works:

  1. For each document group, merge all page texts into one continuous string
  2. Record the page range (e.g., "4-6")
  3. Consolidate consecutive skipped pages into range groups with skip reasons
  4. Return structured JSON response

5. Data Structures

# Internal processing model (not exposed in API)
class PageResult:
    page_number: int        # Global page number (1-indexed)
    text: str               # Full OCR text of this page
    first_lines: str        # First 5 lines — used for title detection
    classified_as: str      # Set during Phase 3: doc type, "SKIP", or "CONTINUATION"


# API response models (in models/schemas.py)

class ExtractedDocument(BaseModel):
    """One complete customs document extracted from the PDF."""
    document_type: str      # "Bill of Lading (BOL)", "Invoice", etc.
    page_range: str         # "1-3", "4-6"
    text: str               # Full OCR text of all pages in this group

class SkippedSection(BaseModel):
    """Pages that were filtered out as unnecessary."""
    page_range: str         # "9-12"
    reason: str             # "No matching document title" or specific junk type

class SmartExtractionResponse(BaseModel):
    """Complete Agent 1 response."""
    total_pages: int        # Total pages in the uploaded PDF
    relevant_pages: int     # Pages that belong to necessary documents
    skipped_pages: int      # Pages that were filtered out
    documents: List[ExtractedDocument]
    skipped_documents: List[SkippedSection]

6. Core Function Implementations

6.1 Phase 1+2: _extract_per_page_text()

def _extract_per_page_text(file_bytes: bytes, mime_type: str) -> List[PageResult]:
    """
    Sends PDF to Document AI in parallel 5-page chunks,
    then extracts per-page text from the response using textAnchor offsets.
    """
    # 1. Split PDF into 5-page chunks
    chunks = split_pdf_to_chunks(file_bytes, pages_per_chunk=5)
    
    # 2. Process chunks in parallel (3 workers, with retry)
    results = [None] * len(chunks)
    with ThreadPoolExecutor(max_workers=3) as executor:
        futures = {
            executor.submit(_process_chunk_docai, chunk, mime_type): i
            for i, chunk in enumerate(chunks)
        }
        for future in as_completed(futures):
            idx = futures[future]
            results[idx] = future.result()
    
    # 3. Extract per-page text from each response
    all_pages = []
    global_page_num = 0
    for chunk_response in results:
        full_text = chunk_response.document.text
        for page in chunk_response.document.pages:
            global_page_num += 1
            page_text = _get_page_text(page, full_text)
            lines = page_text.strip().split('\n')
            all_pages.append(PageResult(
                page_number=global_page_num,
                text=page_text,
                first_lines='\n'.join(lines[:5])
            ))
    
    return all_pages

6.2 Phase 3: _detect_page_title()

TITLE_PATTERNS = {
    "Bill of Lading (BOL)": [
        r"BILL\s+OF\s+LADING", r"\bB/L\b", r"OCEAN\s+BILL",
        r"MASTER\s+BILL", r"HOUSE\s+BILL", r"SEA\s+WAYBILL"
    ],
    "Invoice": [
        r"COMMERCIAL\s+INVOICE", r"\bINVOICE\b",
        r"PROFORMA\s+INVOICE", r"TAX\s+INVOICE"
    ],
    "Packing List": [
        r"PACKING\s+LIST", r"PACKING\s+SLIP",
        r"PACKING\s+NOTE", r"WEIGHT\s+LIST"
    ],
    "Freight Certificate": [r"FREIGHT\s+CERTIFICATE", r"FREIGHT\s+MEMO"],
    "Insurance Certificate": [
        r"INSURANCE\s+CERTIFICATE",
        r"CERTIFICATE\s+OF\s+INSURANCE"
    ],
}

JUNK_PATTERNS = [
    r"COVER\s+LETTER", r"\bFAX\b", r"TRANSMITTAL",
    r"COMPLIANCE", r"TERMS\s+AND\s+CONDITIONS", r"BANK\s+DETAILS"
]

def _detect_page_title(first_lines: str) -> Optional[str]:
    """Pure regex — zero API calls, ~0.01ms per page."""
    text_upper = first_lines.upper()
    for doc_type, patterns in TITLE_PATTERNS.items():
        if any(re.search(p, text_upper) for p in patterns):
            return doc_type
    if any(re.search(p, text_upper) for p in JUNK_PATTERNS):
        return "SKIP"
    return None  # No title → continuation page

6.3 Phase 3+4: _group_and_filter_pages()

def _group_and_filter_pages(
    pages: List[PageResult]
) -> Tuple[List[ExtractedDocument], List[SkippedSection]]:
    """
    Sequential scan — group pages by document type.
    Title found → start new group
    No title → continuation of previous group
    Junk title → skip, break continuation chain
    """
    documents = []
    skipped = []
    current_group = None

    for page in pages:
        title = _detect_page_title(page.first_lines)

        if title and title != "SKIP":
            # Flush previous group
            if current_group:
                documents.append(_merge_group(current_group))
            current_group = {
                "type": title,
                "pages": [page],
                "start": page.page_number
            }

        elif title == "SKIP":
            # Flush and break chain
            if current_group:
                documents.append(_merge_group(current_group))
                current_group = None
            skipped.append(SkippedSection(
                page_range=str(page.page_number),
                reason="Unnecessary document detected"
            ))

        else:  # None — continuation
            if current_group:
                current_group["pages"].append(page)
            else:
                skipped.append(SkippedSection(
                    page_range=str(page.page_number),
                    reason="No document title found"
                ))

    # Flush final group
    if current_group:
        documents.append(_merge_group(current_group))

    return documents, skipped


def _merge_group(group: dict) -> ExtractedDocument:
    """Merge consecutive pages in a group into one document."""
    combined_text = "\n".join(p.text for p in group["pages"])
    end_page = group["pages"][-1].page_number
    return ExtractedDocument(
        document_type=group["type"],
        page_range=f"{group['start']}-{end_page}" if end_page != group["start"] else str(group["start"]),
        text=combined_text
    )

9. Risk Assessment & Mitigation

#RiskSeverityMitigation
1 15-page sync limit exceeded Medium 5-page chunking keeps every request safely under limit. No action needed.
2 20 MB file size exceeded High Chunking splits before sending — each 5-page chunk ≈1-3 MB. For extremely dense scans, auto-reduce to 3-page chunks if any chunk exceeds 18 MB.
3 120 pages/min quota exhausted Medium Queue requests with exponential backoff on 429. Request quota increase via GCP Console if traffic grows.
4 Title regex misses unusual format Medium Safe default: untitled pages following a relevant page are kept as continuations. No data loss occurs — worst case is including some unnecessary continuation pages.
5 Low-resolution scan produces poor OCR Medium Require clients to submit 300+ DPI scans. Cannot be mitigated in software.
6 GCP 503 Service Unavailable Low Retry with exponential backoff (2, 4, 8 seconds). Max 3 retries per chunk.
7 Document AI completely unavailable Low Return HTTP 503 with clear error message. No Gemini fallback — this is intentional to protect Agents 2–5 quota.
